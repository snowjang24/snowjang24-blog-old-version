CSS 방법론 - BEM에 대하여

BEM 소개

2019 프론트앤드 개발 로드맵을 보면 CSS아키텍쳐로 BEM을 추천하고 있다(여기서 추천하는 기술은 많이 쓰이고 있는 기술들이라, 한 번 전체적으로 공부해보는 것이 좋을 것 같다).

2017년에는 추천하는 기술에 포함되지 않았던 BEM은 2018년에 들어 추천하는 기술에 올랐고, 2019년에는 여전히 추천하는 기술에 포함돼있지만 CSS in JS)의 등장으로 금방 자리를 내어줄 것 같다.

CSS in JS는 React같은 Front-End Framework에 최적화되어 있다.

이런 프레임워크의 사용이 주를 이루는 요즘, 금방 BEM을 사용하지 않게 될 것 같다는 생각이 들었다.

추후에 CSS in JS에 관해서도 공부해봐야겠다.

하지만 위의 사진에도 있듯이 이런 방법론이 있다는 것에 의미를 두고 간단히 정리를 해보려 한다.


BEM 이란?

BEM은 Block, Element, Modifier의 약자로 컴포넌트 기반의 웹 개발
